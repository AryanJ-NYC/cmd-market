# Docs Index

This repository uses a harness-style documentation system: short entrypoints route readers to focused source-of-truth docs instead of duplicating everything everywhere.

## Core Docs

- `../README.md`: quick human onboarding and main commands
- `../AGENTS.md`: short routing file for coding agents
- `../ARCHITECTURE.md`: durable technical structure and repo boundaries
- `../apps/web/docs/design.md`: durable visual system, landing-page language, and composition rules
- `audience.md`: durable audience model, personas, and messaging priorities
- `product.md`: durable product truth and current scope
- `development.md`: development workflows, commands, and doc maintenance rules
- `testing.md`: verification strategy and testing expectations

## Planning

- `plans/active/`: significant in-progress work
- `plans/completed/`: completed plans and design notes that still carry decision value

## References

- `references/`: durable summaries of outside material worth keeping in-repo

## Promotion Rule

If a plan or note explains a lasting decision, promote it into a durable doc or archive it in `plans/completed/`. If it is only temporary thinking, keep it out of git.
